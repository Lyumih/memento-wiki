# Дизайн: страница «Эмуляция карты» (`memento-card-emulation`)

**Дата:** 2026-03-31  
**Статус:** утверждено в сессии brainstorming (v1 = песочница текста; связь с `memento-roll` = вариант **A**)  
**Репозиторий:** `gen-memento-docs`  
**Связанные документы:** `content/dev/memento-card-emulation.mdx`, `content/dev/memento-roll.mdx`, `src/memento/rollCardLevelUp.ts`, `src/memento/resolvePercentToken.ts`, `src/widgets/MementoRollLab.tsx`

## 1. Цель

Страница разработчика **`content/dev/memento-card-emulation.mdx`** даёт **песочницу** для проверки произвольного текста (как черновик описания умения, карты или предмета): как при разных уровнях карты **`L`** выглядят числа после подстановки токенов Memento Mori вида **`%%`**.

На странице **`/dev/memento-roll`** остаются **норматив формул**, **график `E₁(L)`**, **таблицы `E_cum(S, T)`** и детальная связка с правилами Gen. Страница «Эмуляция карты» **не дублирует** график и таблицы; вводный текст со **ссылкой** на `/dev/memento-roll` обязателен.

## 2. Объём v1 и вне объёма

**В v1:**

- Многострочный ввод текста (textarea), отображение с **`white-space: pre-wrap`** (как осмысленный аналог переносов в YAML `body`).
- Эмуляция уровня карты: параметр старта **`S`** (`1…999`, слайдер + числовой ввод), текущий **`L`**, кнопки «Поднять уровень» / «Сбросить уровень», отображение последнего **`r`** и исхода броска — **то же правило**, что **`rollCardLevelUp`** и тот же UX-паттерн, что в **`MementoRollLab`** (сброс **`L := S`**).
- Блок **превью**: исходный текст, в котором каждое **непересекающееся** вхождение токена в форматах **`BASE%%`**, **`BASE%%CAP`**, **`BASE%%-P`** (семантика как **`parsePercentToken` / `resolvePercentValue`**) заменено на вычисленное **целое** при текущем **`L`**.

**Вне v1:**

- Выбор сущности из каталога БД, привязка к YAML-записям.
- «Открытие модификаторов» по уровню и любая новая схема данных под моды.
- Персистентность черновика (**`localStorage`** и т.п.) — только отдельной спецификацией при необходимости.

## 3. Поведение превью и ошибок

- **Норматив подстановки:** для каждого распознанного токена — **`resolvePercentValue(L, token)`** (или эквивалент без изменения формул).
- **Невалидные фрагменты:** подстроки, которые **не** проходят **`parsePercentToken`**, **не заменять** (остаются в превью как в исходном тексте).
- **Опционально в v1:** под превью компактный список предупреждений (например «похоже на токен, но не распознан») — **не обязателен**; если реализован, критерии должны быть однозначны (например только явные совпадения по тому же языку, что сканер токенов), без эвристик «угадывания».

- **`L` в UI:** после сброса **`L ≥ 1`**; диапазон **`S`** согласован с лабораторией (**`1…999`**). Для **`resolvePercentValue`** при **`level < 0`** — поведение как в существующем коде (не используется в нормальном сценарии UI).

## 4. Реализация (модуль и UI)

### 4.1 Сканирование и замена токенов в строке

В репозитории **нет** готовой функции «подставить все токены в произвольной строке». В **`src/memento/resolvePercentToken.ts`** (или рядом, с реэкспортом при необходимости) добавить **чистую функцию**, например **`replacePercentTokensInText(level: number, text: string): string`**, которая:

- находит в **`text`** все непересекающиеся вхождения шаблона, **эквивалентного** грамматике **`parsePercentToken`** (plain / cap / neg);
- для каждого успешного разбора подставляет строковое представление результата **`resolvePercentValue`** (при **`null`** — оставляет исходную подстроку или помечает для предупреждения — **зафиксировать один вариант в реализации**, предпочтительно **оставить исходник** при **`null`**).

Добавить **юнит-тесты**: несколько токенов в одной строке, текст без токенов, смесь валидных и невалидных подстрок, граничные уровни **`L`** согласованно с существующими тестами **`resolvePercentValue`**.

### 4.2 Виджет страницы

- Новый React-компонент (рабочее имя **`CardEmulationSandbox`** или близкое), подключённый из **`memento-card-emulation.mdx`** через **`MdxShell`** / существующий механизм виджетов (как **`MementoRollLab`** на `memento-roll`).
- **Рекомендация brainstorming (v1):** **не** выносить общий «пульт L» из **`MementoRollLab`** в этой итерации (допустимо дублирование небольшого блока UI); вынесение общего компонента — отдельный рефакторинг после появления боли от дублирования.

## 6. Критерии готовности

- На **`/dev/memento-card-emulation`** видны ввод, эмуляция **`L`** (как описано), превью с подстановкой всех валидных токенов в тексте.
- Ссылка на **`memento-roll`** присутствует; график и таблицы **только** на `memento-roll`.
- Поведение броска уровня совпадает с **`rollCardLevelUp`**; токены — с **`resolvePercentValue`**.
