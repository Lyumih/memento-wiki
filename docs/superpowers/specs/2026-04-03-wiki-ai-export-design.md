# Экспорт вики для ИИ и IDE: один `.md`, JSON БД, страница на сайте

## 1. Цели

- Дать **один скачиваемый Markdown-файл** со всеми текстовыми разделами вики, пригодный для открытия в IDE в режиме предпросмотра и для передачи в ИИ.
- Дать **один JSON-файл** с полным каталогом сущностей БД (как сейчас формируется `db.json`), без дублирования тысяч записей внутри `.md`.
- На **сайте**: **предпросмотр** содержимого того же `.md` (рендер в HTML) и **кнопки скачивания** `.md` и `.json`.

## 2. Артефакты и имена

| Артефакт | Назначение | Поставка |
|----------|------------|----------|
| `memento-wiki-export.md` | Все статьи (из MDX) в одном файле | Статика в `public/` после prebuild; тот же контент дублируется в `src/generated/` при необходимости для импорта `?raw` в бандле |
| `memento-db.json` | Полный снимок БД для машин и ссылок по `id` | Копия содержимого собранного `db.json` в `public/` под этим именем |

Имена файлов фиксированы в спеке, чтобы ссылки на скачивание и документация не расходились с реализацией.

## 3. Содержимое `memento-wiki-export.md`

**Источник:** все файлы `content/**/*.mdx`, **кроме** дерева `content/db/**` (каталог БД в YAML остаётся только в JSON).

**Порядок разделов:**

1. `content/index.mdx` (главная) — первой.
2. Далее группы: `players` → `dev` → `games` (как в логике навигации).
3. Внутри группы: сортировка по полю frontmatter `order` (число), при равенстве — по относительному пути (`localeCompare`, `ru`).

**Формат каждой секции:**

- Заголовок второго уровня в одной строке: **`## {title} (`{urlPath}`)`** — сначала человекочитаемый `title`, затем путь в круглых скобках в бэктиках (уникальность якорей при совпадении названий страниц).
- Краткая метастрока: путь к исходнику в репозитории, например `content/...`.
- Тело: содержимое MDX **без** frontmatter (как в исходнике после `---`). Фрагменты с JSX/React-компонентами (`<RollLevelDemo />` и т.д.) в чистом Markdown Preview остаются **исходным текстом** — это ожидаемое ограничение v1; в начале файла — короткое предупреждение об этом.

**Вступление в начале файла:**

- Назначение файла, связь с `memento-db.json`, предупреждение про виджеты.
- **Оглавление** со ссылками на якоря секций; алгоритм slug для ссылок должен совпадать с тем, что использует рендер предпросмотра на сайте (например общий подход на базе slug от строки заголовка `## {title} (`{urlPath}`)`).

**Блок про БД внутри `.md` (не дублировать весь каталог):**

- Краткое текстовое описание схемы (типы сущностей, основные поля, где лежит полный дамп).
- **Около 10 примеров** записей (укороченные или полные — по усмотрению реализации), каждый с явной отсылкой к полному JSON: тип коллекции (`items` / `skills` / `modifiers`) и `id`, чтобы по файлу можно было найти запись в `memento-db.json`.

## 4. Сайт: страница экспорта

- **Маршрут:** `/export` (или эквивалент с тем же смыслом; в спеке зафиксировано **`/export`**).
- **Содержимое страницы:**
  - Заголовок и краткое описание.
  - **Предпросмотр:** HTML из того же текста, что в `memento-wiki-export.md` (Markdown → HTML с поддержкой GFM по необходимости; заголовки получают стабильные id, согласованные с оглавлением в файле).
  - **Скачать** `memento-wiki-export.md`.
  - **Скачать** `memento-db.json`.
- **Обнаружение:** ссылка «Экспорт» в шапке рядом с навигацией (или в футере области контента), **без** обязательного добавления пункта в автоматически собираемое меню из `nav.json`, чтобы не смешивать контентные страницы с инфраструктурным маршрутом.

## 5. Сборка

- Генерация `memento-wiki-export.md` и копирование JSON в `public/` выполняются в рамках существующего пайплайна **prebuild** (после успешного построения `nav.json` и `db.json` и прохождения проверок ссылок, либо в том же проходе, где это безопасно по порядку зависимостей).
- При ошибке prebuild артефакты экспорта не обновляются (как и остальные generated-файлы).

## 6. Хостинг (SPA)

- Убедиться, что `memento-wiki-export.md` и `memento-db.json` из каталога статики отдаются **как файлы**, а не перехватываются правилом «всё на `index.html`». Для Vercel файлы из `public` обычно имеют приоритет над rewrites; это поведение зафиксировать в проверке перед релизом.

## 7. Нефункциональные ограничения v1

- Очень большой `.md` может замедлять предпросмотр в IDE и в браузере — допускается короткое предупреждение на странице `/export`.
- Экспорт не обязан воспроизводить интерактив виджетов; только текст и разметка из исходников.

## 8. Объём и границы

- **Входит:** описанные артефакты, prebuild, страница предпросмотра и скачивания, навигационная ссылка.
- **Не входит в v1:** отдельный PDF, XML, RSS, версионирование экспорта по дате в URL, локализация имён файлов.

## 9. Тестирование (ориентиры)

- Юнит- или интеграционная проверка скрипта сборки: непустой вывод, наличие оглавления, минимум одной секции при наличии MDX, корректный порядок (индекс первым).
- Скриншотный или ручной чек: страница `/export` отображает текст, ссылки скачивания отдают ожидаемые файлы; после деплоя — прямой GET к `.md` и `.json` возвращает не HTML-оболочку SPA.
