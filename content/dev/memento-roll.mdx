# Формулы: бросок уровня карты (Memento)

Правило в коде: `rollCardLevelUp(currentLevel, r)`; `r` — равномерное целое от **1** до **100** включительно.

Условие успеха броска (шанс повышения за одну попытку задаётся этой логикой; исходник — `src/memento/rollCardLevelUp.ts`):

```typescript
export function rollCardLevelUp(
  currentLevel: number,
  randomInt1to100: number,
): boolean {
  const r = randomInt1to100
  if (currentLevel > 100) return r === 100
  return r === 100 || r >= currentLevel
}
```

- При **L > 100** успех только если **r = 100**, то есть **P(L) = 1/100**, ожидание числа бросков до первого успеха **E₁(L) = 100**.
- При **1 ≤ L ≤ 100** успех при **r = 100 или r ≥ L** (как в исходнике). Для таких **r** это ровно **101 − L** благоприятных исходов, **P(L) = (101 − L) / 100**, **E₁(L) = 1 / P(L)**.

**Кумулятив** от старта **S** до цели **T** (каждый успех даёт +1 уровня, броски независимы в этой модели):

- если **T ≤ S**, ожидаемое число бросков **0**;
- если **T > S**: **E_cum(S, T) = Σ<sub>L = S</sub><sup>T−1</sup> E₁(L)** (вещественное значение; в таблицах выше оно округляется только для отображения).

# Формулы: токены `%%`

Поведение совпадает с `resolvePercentToken` / `resolvePercentValue` в этом репозитории (спека §3 вики, приложение B).

Строка в формате **BASE%%**, **BASE%%f**, **BASE%%CAP** или **BASE%%-P** (см. парсер в коде). Для уровня карточки **L ≥ 0**:

- **plain** (`BASE%%`): значение **round(BASE × (1 + 0,01·L))** (`Math.round`).
- **plain floor** (`BASE%%f`): тот же множитель **(1 + 0,01·L)**, значение **floor(BASE × (1 + 0,01·L))** (`Math.floor`) — удобно для счётчиков (например снарядов), чтобы при промежуточных **L** не «подпрыгивало» вверх раньше порога.
- **cap** (`BASE%%CAP`): **t = min(L, 100)**, значение **round(BASE × (1 + (CAP/100)·(t/100)))**; при **L > 100** масштаб по уровню заморожен на **t = 100** (решение v0).
- **neg** (`BASE%%-P`, P > 0): **t = min(L, 100)**, значение **round(BASE × (1 − (t/100)·(P/200)))**; при **L > 100** — та же заморозка по **t**.

# Уровень модификатора **Lm** (встроенные моды на карте)

Для **встроенных** модификаторов у карты умения, предмета и т.д. вводится отдельный целочисленный уровень **`Lm`** на каждый слот. **Один бросок** на повышение **`Lm`** использует **то же условие успеха**, что и `rollCardLevelUp(currentLm, r)` выше (тот же закон, что для уровня карты **`L`**). Очереди бросков и игровые события для **`L`** и для **`Lm`** **разделены**: прокачка карты и прокачка модификаторов независимы.

Слот для выбора нового модификатора открывается при порогах по **`L`** (1-й при **L ≥ 75**, далее +100 за каждый следующий слот). Подробности, оффер из трёх вариантов и порядок применения эффектов — на странице [Модификаторы карт, умений и предметов](/dev/memento-modifiers).
